Improving map performance with Canvas

Adam Mark

Background

From 2020 to 2023, I developed dashboards and mapping tools for a warehouse automation system using TypeScript, React and SVG.

Problem

Our custom mapping system was built with SVG. While SVG was both flexible and easy to use—especially with React—it started to drag when we rendered more than few hundred elements on the map. This prevented us from creating complex data visualizations, such as heat maps.

Solution

I created a drawing canvas using the Canvas API, along with utility methods for working with the map's coordinate space.

The canvas layer enabled all kinds of arbitrary data visualizations, such as the one shown here for demonstration purposes, in which I plotted 1,000 random, color-coded¹ bubbles:

As the user pans and zooms, the canvas redraws itself up to 50 times a second with no appreciable impact on performance.

The code

First, I created a hook in the map library that allowed clients to access the drawing context:

const canvas = useMapCanvas();
const context = canvas.getContext('2d');

Then I added a callback function to the map component, allowing clients to respond to zoom and pan events:

<Map
  mapStack={...}
  onZoom={(evt) => ...}
/>

Finally, for this demo, I created 1,000 random bubbles and a function to render them whenever the map is manipulated. Putting it all together:

import { Map, useMapCanvas, resetMapContext } from 'map-element';

function DemoMap() {
  const canvas = useMapCanvas();
  const context = canvas.getContext('2d');
  const bubbles = ...;

  function draw(context: CanvasRenderingContext2D, scale: number, center: Point, size: Dimensions) {
    requestAnimationFrame(() => {
      resetMapContext(context, scale, center, dimensions, true);
  
      bubbles.forEach((bubble) => {
        context.fillStyle = bubble.color;
        context.beginPath();
        context.arc(bubble.x, bubble.y, bubble.radius, 0, 2 * Math.PI);
        context.fill();
      });
    });
  }

  return (
    <Map
      mapStack={...}
      onZoom={(evt) => draw(context, evt.scale, evt.center, evt.size)}
    />
  );
}

Conclusion

Adding a low-level drawing layer to the map opened the door to all kinds of interesting (and fast) data visualizations.

¹ The rainbow effect is courtesy of ChatGPT. I asked it to create a utility function that accepts an input x on a scale of 0 to 1 and returns a corresponding color of the rainbow (as a hex string) on a scale of red to violet. Its first attempt failed, but it got there with a little coaching.
